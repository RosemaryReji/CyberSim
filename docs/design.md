# CYBERSIM DESIGN SYSTEM DOCUMENT

*(Inspired by darkstarlabs.io, vladburca.com, skylights.tech)*

## 1. Core Design Philosophy

"Make complex systems feel like interactive experiences, not
interfaces."

CyberSim should feel like a simulation lab or control panel, not a
traditional website.

## 2. Visual Identity

Mood: Dark, futuristic, technical, immersive\
Keywords: Simulation, system interface, cyber lab

## 3. Color System

Background: #050505\
Surface: #0B0F14\
Card: #11161C

Primary Accent: #00F0FF\
Secondary Accent: #7A5FFF

Success: #00FF9C\
Warning: #FFC857\
Danger: #FF3B3B

## 4. Typography

Headings: Orbitron / Space Grotesk\
Body: Inter / Sora\
Code: Monospace

Rules: - Large bold headings\
- Tight spacing\
- Minimal body text

## 5. Layout

-   Fullscreen sections\
-   Scroll storytelling\
-   Large spacing

Sections: 1. Hero (immersive intro) 2. Modules (attack cards) 3.
Simulation UI 4. Feedback panel

## 6. Components

### Buttons

-   Transparent
-   Neon border
-   Glow on hover

### Cards

-   Dark background
-   Soft shadow

### Inputs

-   Terminal style
-   Monospace font

### Panels

-   Grid lines
-   Live updates

## 7. Motion & Animation

-   Scroll animations\
-   Hover glow effects\
-   Terminal-style logs

Example: \> Initializing attack... \> Injecting payload... \>
Vulnerability detected!

## 8. Module UI

Phishing: - Fake email interface

XSS: - Code editor layout

Brute Force: - Progress visualization

## 9. UX Principles

-   Show, don't tell\
-   Guided exploration\
-   Immediate feedback

## 10. Design Identity

"A futuristic cyber warfare training console disguised as a web app"

## 11. Tech Stack

Frontend: Next.js, Tailwind\
Animation: Framer Motion, GSAP\
3D (optional): Three.js

## 12. Differentiation

Not a basic dashboard --- an immersive simulation experience.
